# План оптимизации сайта для ИИ-агентов (AI-SEO / AIO)

Этот план содержит шаги по превращению `alexgetman.com` в максимально дружелюбный ресурс для ИИ-ассистентов (ChatGPT, Claude, Perplexity, Gemini, Apple Intelligence), что повысит вероятность попадания ваших постов в их ответы пользователям.

---

## 1. Текущий статус (Что уже работает)
* **Доступность**: В `robots.txt` открыт полный доступ (`Allow: /`) для всех краулеров.
* **Content-Signal**: В `robots.txt` прописана директива `Content-Signal: ai-train=no, search=yes, ai-input=no` (запрет обучения на текстах, но разрешение на поиск и цитирование в ответах).
* **Служебный файл `llms.txt`**: В корне сайта отдает структурированный список последних постов и соцсетей в Markdown-формате.
* **Семантическая разметка Schema.org**: Разметка `Person` с привязкой к социальным сетям повышает E-E-A-T авторитетность автора в глазах поисковых моделей.

---

## 2. Предлагаемые улучшения

### Шаг 1. Генерация Markdown-версий для всех постов (`.md` endpoint)
ИИ-модели работают с текстовыми токенами. Чтение HTML-версии страницы тратит много лишних токенов на теги разметки, стили и скрипты. Чистый Markdown — идеальный формат для LLM.

* **Реализация**:
  * Создать динамический роут в Astro для отдачи постов в Markdown: `src/pages/[postId]/[slug].md.ts` (или `.md.astro`).
  * При обращении по адресу `https://alexgetman.com/{id}/{slug}.md` отдавать только чистый текст статьи с заголовками в формате Markdown.
  * Обновить генератор `llms.txt`, чтобы ссылки в списке постов вели на их `.md` версии.

### Шаг 2. Расширенный фид для ИИ-моделей (`/feed-ai.json`)
Стандартные RSS-ленты содержат полный текст или краткий анонс. Для ИИ-моделей можно отдавать более структурированные данные.

* **Реализация**:
  * Создать эндпоинт `/feed-ai.json`.
  * Для каждого поста добавить метаинформацию:
    * `tldr` или `summary`: сжатое резюме поста (1-2 предложения).
    * `key_entities`: список упоминаемых технологий, нейросетей, компаний.
    * `actions`: ссылки на внешние ресурсы (GitHub, оригинальный твит и т.д.).

### Шаг 3. Аналитика и мониторинг активности ИИ на дашборде
Сейчас мы видим только общую посещаемость. Было бы ценно видеть, какие именно посты вызывают интерес у ИИ-ассистентов.

* **Реализация**:
  * В сборщике метрик Nginx логов на сервере отфильтровать запросы от ботов с User-Agent: `OAI-SearchBot` (поиск ChatGPT), `GPTBot` (OpenAI), `ClaudeBot` (Anthropic), `Perplexibot` (Perplexity).
  * Сохранять количество запросов от ИИ-агентов к каждому посту в базу `pipeline.db` (новая метрика `ai_hits`).
  * Вывести эту метрику в Command Center отдельной колонкой (например, 🤖 `AI Hits`). Это позволит видеть, какие темы вызывают наибольший интерес у пользователей нейросетей.

### Шаг 4. Автоматическая генерация тегов `alt` для изображений
Для классического SEO (включая поиск по картинкам Google/Яндекс) изображения должны иметь описательные теги `alt`. Так как в Telegram-постах вы не пишите описания медиафайлов вручную, мы можем автоматизировать это.

* **Реализация**:
  * В шаблоне Astro (`[slug].astro`) при рендере тегов `<img>` автоматически подставлять заголовок поста (`postTitle`) в атрибут `alt`.
  * Если в посте содержится несколько изображений (галерея), генерировать значения динамически: `${postTitle} - фото ${index + 1}`.
  * Это улучшит общую валидность структуры DOM для поисковых ботов без каких-либо ручных действий с вашей стороны.

### Шаг 5. Поддержка Markdown Negotiation (Accept: text/markdown)
Аудит `isitagentready.com` показал, что сайт должен возвращать Markdown-версию ответа, если ИИ-клиент запрашивает её через заголовок `Accept: text/markdown`.

* **Реализация**:
  * В Nginx на сервере настроить обработку заголовка `Accept`: если пришел `Accept: text/markdown`, делать внутренний редирект с `/` на `/index.md`, а с `/{postId}/{slug}/` — на `/{postId}/{slug}.md`.
  * Возвращать правильный заголовок `Content-Type: text/markdown` для таких ответов.

### Шаг 6. Отдача HTTP Link Headers
ИИ-агенты ищут служебную разметку через заголовки ответов (RFC 8288). 

* **Реализация**:
  * Прописать в конфигурации Nginx отдачу заголовка `Link` для главной страницы:
    `Link: </.well-known/api-catalog>; rel="api-catalog", </.well-known/agent-skills/index.json>; rel="agent-skills", </llms.txt>; rel="service-doc"`

### Шаг 7. DNS для обнаружения ИИ (DNS-AID)
Публикация специальных DNS-записей типа SVCB/HTTPS (Service Mode) на домене для автоматического поиска MCP/API-каталогов роботами.

* **Реализация**:
  * Добавить в Cloudflare DNS записи:
    * `_index._agents.alexgetman.com`
    * `_mcp._agents.alexgetman.com`
    С указанием параметров alpn и эндпоинтов, ссылающихся на ваши API-каталоги.

## 3. Ожидаемый эффект
* Повышение скорости индексации новых постов ИИ-моделями (модели будут скачивать статьи в виде Markdown за миллисекунды).
* Увеличение частоты упоминаний в ответах ChatGPT Search и Perplexity за счет готовой структурированной выжимки (TL;DR) в фиде.
* Точное понимание интересов ИИ-аудитории через дашборд.
